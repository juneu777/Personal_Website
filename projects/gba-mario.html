GBA Super Mario (C)

Software Engineer • Atlanta, GA • Jan–Feb 2024

Overview Tech & Links Architecture

Overview

  • Developed an interactive Super Mario–style game in C targeting a Game Boy Advance emulator.
  • Implemented Mode 3 video rendering and used DMA transfers for efficient image blits to ensure smooth gameplay without tearing.
  • Built 2D movement for Mario with responsive button inputs and collision detection for tiles/objects.

Tech Stack

  • Language: C
  • Platform: Game Boy Advance (Mode 3)
  • Graphics: Framebuffer rendering, DMA for fast memory copies
  • Input: GBA keypad polling for A/B/Start/Select/D-Pad
  • Core systems: game loop, sprite/tile handling, basic physics & collisions

Add repo / demo links when ready.

Architecture

  • Game Loop: update → render → VBlank sync for stable frame pacing.
  • Rendering (Mode 3): direct pixel writes to the framebuffer; DMA used for large blits (backgrounds/sprites) to minimize CPU time and tearing.
  • Input Handling: polled keypad state with debouncing for smooth controls.
  • World & Physics: tile-based map; AABB-style collision checks for ground, walls, and hazards.
  • Entities: Mario player struct (position, velocity, state); simple enemy/obstacle structs as needed.
